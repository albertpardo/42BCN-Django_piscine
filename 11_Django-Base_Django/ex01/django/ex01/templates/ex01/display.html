Ex01: Display process of a static page.

Displaying a static web page through Django involves a structured flow that transforms a user's request into a rendered HTML document. Here is the step-by-step process:

  1. The Request: It all starts when a user enters a URL (e.g., /ex01/display) in their browser. This sends an HTTP Request to the web server.
  2. URL Dispatcher (urls.py): Django receives the request and consults the URLconf. It searches for a pattern that matches the requested URL to determine which View function should handle it.
  3. The View (views.py): The view acts as the "controller." For a static page, the view typically calls the render() function. This function links the logic with the presentation.
  4. The Template Engine: The engine looks for the specific HTML file. If the template uses inheritance (like base.html), the engine merges the "skeleton" with the child's specific blocks to build a final, complete HTML file.
  5. The Response: The view returns an HttpResponse object containing the generated HTML. Django sends this back to the user's browser, which finally renders the page on the screen.

This architecture follows the Separation of Concerns principle, keeping your logic (Python), data, and presentation (HTML) organized and independent.
